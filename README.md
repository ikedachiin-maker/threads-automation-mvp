# Threads Automation MVP

> Claude Code使ったThreads完全自動化システム - MVP版

## 概要

このプロジェクトは、Threads（Instagram）の投稿を**AI（Claude）で完全自動化**するシステムです。
シロウ氏（[@shiro_life0](https://x.com/shiro_life0)）の記事「Claude code使ったら完全自動で3万稼げてた件」を参考に、Claude Codeで実装しています。

### 実現すること

- ✅ AIによる投稿自動生成（5-10本/バッチ）
- ✅ 品質スコア自動採点（7.0/10.0未満は自動棄却）
- ✅ 類似度チェック（過去投稿と0.85以上類似なら棄却）
- ✅ 投稿パターンの自動ローテーション（15種類以上）
- ✅ Threads API経由での自動投稿
- ✅ 人間っぽい投稿間隔（最低1時間、1日10スロット）

### MVP範囲（このバージョン）

**含まれるもの:**
- ライター（投稿生成AI）
- 品質スコア自動採点
- 類似度チェック
- 投稿キュー管理
- Threads API投稿機能

**後で追加するもの:**
- リサーチャー（YouTube/Xからネタ収集）
- アナリスト（パフォーマンス分析）
- フェッチャー（メトリクス自動取得）
- スーパーバイザー（異常検知・監視）
- 完全自動実行（cron/launchd）

## 6つのAIエージェント設計（フルバージョン）

このシステムは最終的に6つの専門AIエージェントで構成されます：

### 1. リサーチャー（ネタ収集担当）
- YouTube/Xから最新ネタ自動収集
- テーマツリー参照して不足分を補充
- 文字起こし解析
- JSONで出力

### 2. アナリスト（分析担当）
- 過去投稿のパフォーマンス分析
- エンゲージメント率計算
- 伸びたパターン抽出
- ライターへのフィードバック生成

### 3. ライター（投稿作成担当）⭐ MVP実装済み
- リサーチデータ読み込み
- 15種類以上の投稿パターンをローテーション
- 1行目パターン集（265件）参照
- 品質自己採点（10項目×10点）
- 過去100件との類似度チェック
- 5-10本バッチ生成

### 4. ポスター（投稿実行担当）⭐ MVP実装済み
- 投稿キューから取得
- Threads API経由で投稿
- コメント欄への続き/リンク配置
- 投稿間隔制御（最低1時間）
- 投稿履歴記録

### 5. フェッチャー（データ取得担当）
- 24時間後メトリクス取得
- 閲覧数・いいね・リプライ取得
- 投稿履歴に追記

### 6. スーパーバイザー（監視担当）
- エラー監視（3回連続で停止）
- 投稿予定チェック
- KILL_SWITCH（緊急停止）
- 通知機能

## 設計の4つの原則

記事で強調されていた設計原則：

### 1. エージェント分離
1エージェント = 1タスク。複数の仕事をさせるとパンクする。

### 2. ナレッジとロジックの分離
口調、NGワード、投稿パターン、ターゲット情報は全て外部ファイル（`knowledge/`）に切り出し。
コードに人格をハードコードしない。

### 3. 状態管理
投稿履歴、キュー、分析結果を全てJSONで管理。
AIが「昨日何を投稿したか」を記憶している状態を維持。

### 4. 安全装置
- 品質スコア7.0未満は自動棄却
- 過去投稿との類似度0.85以上も棄却
- 1日の投稿上限15件
- 最低投稿間隔1時間
- KILL_SWITCH（緊急停止）

## 重要な失敗談と対策

記事で紹介された2つの大事故：

### 事故1: 1分で10連投してアカウント凍結
**原因**: 投稿間隔設定ミス
**対策**:
- 最低1時間間隔
- 1日10スロット（8時〜25時）
- 人間のリズムに合わせる

### 事故2: 同じ投稿を30回ループ
**原因**: 類似度チェックがなかった
**対策**:
- 過去100件との類似度比較（0.85以上棄却）
- 直近3件と同じパターン禁止
- 15パターン強制ローテーション
- 同テーマ3連続で自動切り替え

## セットアップ

### 1. 環境準備

```bash
# Node.js 18以上推奨
node --version

# プロジェクトクローン
git clone https://github.com/yourusername/threads-automation-mvp.git
cd threads-automation-mvp

# 依存パッケージインストール
npm install
```

### 2. 環境変数設定

`.env.example`をコピーして`.env`を作成：

```bash
cp .env.example .env
```

`.env`ファイルを編集：

```env
# Threads API（Instagram Graph API経由）
THREADS_ACCESS_TOKEN=your_access_token_here
THREADS_USER_ID=your_threads_user_id

# Claude API（投稿生成用）
ANTHROPIC_API_KEY=your_claude_api_key

# OpenAI API（代替手段）⭐
OPENAI_API_KEY=your_openai_api_key

# 使用するAI（claude または openai）⭐
AI_PROVIDER=openai

# 設定
DRY_RUN=true  # trueの間は実際に投稿しない（テスト用）
```

**⭐ OpenAI APIを使う場合**:
- `AI_PROVIDER=openai` に設定
- `OPENAI_API_KEY` に OpenAI APIキーを設定
- GPT-4oが投稿を生成します（品質はClaudeとほぼ同等）

### 3. ナレッジファイル設定

`knowledge/persona.json`を編集してアカウントの人格を設定：

```json
{
  "genre": "転職",
  "tone": "カジュアル・断言型",
  "target": "20-30代転職検討者",
  "expertise": "転職エージェント内部事情",
  "ng_words": ["絶対", "100%", "誰でも"]
}
```

`knowledge/topics.json`にネタを追加：

```json
{
  "topics": [
    {
      "theme": "面接攻略",
      "content": "面接で落ちる人の共通点3つ",
      "keywords": ["面接", "失敗", "準備不足"]
    }
  ]
}
```

**デモ投稿の内容**:
1. 短文完結型 - エージェント手数料の仕組み暴露
2. 需要確認型 - 年収交渉100万UP手法
3. リスト型 - 面接で落ちる人の共通点3つ
4. アンチテーゼ型 - 「3年我慢」の常識を否定
5. 警告型 - 職務経歴書テンプレートの危険性

全て品質スコア8.0以上、類似度チェック済みの実用的な投稿です。

### 通常モード（Claude API使用）

### 投稿生成

```bash
# 5-10本の投稿を生成してキューに追加
node generate.js

# 生成された投稿を確認
cat state/post_queue.json
```

**生成プロセス**:
1. `topics.json`からネタを読み込み
2. Claude APIで投稿を生成

4. 過去投稿との類似度チェック（0.85以上は棄却）
5. 合格した投稿を`post_queue.json`に追加

### 投稿実行

```bash
# キューの先頭を投稿
node post.js

# DRY_RUN=true の場合は投稿プレビューのみ
# DRY_RUN=false にすると実際に投稿される
```

## Threads API について

### 取得方法

1. **Meta Developer登録**
   - https://developers.facebook.com/
   - アプリ作成

2. **Threads API申請**
   - 現在Beta版（2026年3月時点）
   - Instagram Business/Creator アカウントが必要

3. **アクセストークン取得**
   - Instagram Graph API経由
   - スコープ: `threads_basic`, `threads_content_publish`, `threads_read_replies`

### API制限

- 投稿: 250件/日
- レート制限: 200リクエスト/時間
- 画像サイズ: 8MB以下

**参考**: [Threads API Documentation](https://developers.facebook.com/docs/threads)

## 品質スコア採点基準

投稿は以下の10項目で自動採点されます（各10点満点）：

1. **フックの強さ** - 1行目で引き込めるか
2. **有益性** - 読者にとって役立つか
3. **具体性** - 抽象論ではなく具体例があるか
4. **テンポ感** - リズムよく読めるか
5. **ペルソナ一致度** - アカウント人格と一致するか
6. **読みやすさ** - 改行・文字数適切か
7. **アクション誘導** - コメント/いいねを促すか
8. **独自性** - ありきたりではないか
9. **感情共鳴** - 共感を呼ぶか
10. **完結性** - 言いたいことが完結しているか

**合格ライン**: 平均7.0/10.0以上

7.0未満は自動で書き直し（2回まで）。それでもダメなら棄却。

## 投稿パターン（15種類）

`knowledge/post_patterns.json`で定義：

1. **短文完結型** - 1-2行で完結
2. **コメント誘導型** - 質問で終わる
3. **ツリー展開型** - リプライで続く
4. **リスト型** - 箇条書き
5. **暴露型** - 内部事情を暴露
6. **需要確認型** - 「知りたい人いる?」
7. **逆説型** - 常識を覆す
8. **ストーリー型** - 体験談
9. **比較型** - AとBを比較
10. **警告型** - 「〜するな」
11. **ステップ型** - 手順を解説
12. **データ型** - 数字で説得
13. **質問投げかけ型** - 最初から質問
14. **共感型** - 「わかる」を引き出す
15. **アンチテーゼ型** - 反対意見

直近3件と同じパターンは自動で除外されます。

## 類似度チェック

過去100件の投稿と比較して、**コサイン類似度0.85以上**なら自動棄却。

**アルゴリズム**:
1. TF-IDFベクトル化
2. コサイン類似度計算
3. 閾値0.85以上で「類似しすぎ」判定
4. 自動棄却して再生成

これがないと「転職エージェントに騙されるな」を30回ループします（実話）。

## なぜThreadsなのか

記事より引用：

> Xはフォロワーいないと基本誰にも届かない。インスタはリール地獄。TikTokは動画編集。
>
> でもThreadsだけ、**フォロワー0でもテキストだけでバズれる。**
>
> アルゴリズムがフォロワー外にガンガン拡散してくれるので、新規アカウントでもちゃんと伸びる。
>
> で、テキストベースってことは、AIとの相性がえぐい。テキスト生成なんてAIが一番得意なやつ。

## トラブルシューティング

### 投稿が生成されない
→ `topics.json`にネタが入っているか確認
→ Claude API キーが正しいか確認

### 品質スコアが低すぎる
→ `knowledge/persona.json`を調整
→ `topics.json`のネタの質を向上

### 類似度チェックで全部棄却される
→ 投稿パターンを増やす
→ 類似度閾値を0.85 → 0.90に緩める

### Threads API エラー
→ アクセストークンが有効か確認
→ API制限に達していないか確認
